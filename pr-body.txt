### Description

Fixes order response schema validation and enforces a tick-size price floor on option limit orders to prevent trade exit failures.

### Changes

- `src/schemas/smartApi.ts`: Updated `SmartApiOrderResponseSchema` to allow `orderid`, `uniqueorderid`, and `script` to be `null` or `optional()`.
- `src/execution/brokerClient.ts`: Added validation check to ensure `parsed.data.orderid` exists when `parsed.status` is `true`, surfacing exact broker error messages when order placement is rejected.
- `src/execution/executionManager.ts`: Enforced minimum limit price floor of `₹0.05` (`OPTION_TICK_SIZE`) in `placeLimitOrderWithReprice` to prevent zero-price limit order rejections.
